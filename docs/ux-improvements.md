# フォーム登録〜回答フロー UX 改善提案

本提案は、現状の画面構成（ログイン → ダッシュボード → フォーム作成/編集 → 回答画面）を前提に、
「離脱率の低下」「作成効率の向上」「回答完了率の向上」を目的として整理したものです。

## 1. 登録・ログイン導線

- **ログイン失敗時の理由を明示**
  - 現在は汎用メッセージ中心のため、`ユーザーID未入力` / `存在しないID` を区別して表示。
- **ログインフォームに補助情報を追加**
  - 例: 「管理者用IDでログイン」「回答者はログイン不要」を入力欄直下に短く表示。
- **入力保持とフォーカス制御**
  - バリデーションエラー時に入力値を保持し、エラー対象へ自動フォーカス。

## 2. ダッシュボード（一覧）

- **主要CTAの優先順位を明確化**
  - 「新規フォーム作成」を最優先ボタンとして固定表示（スクロール時も見える配置）。
- **フォームカードに状態表示を追加**
  - 例: `公開中/下書き`、`最終更新日時`、`回答件数`。
- **検索/絞り込みの導入**
  - フォーム数増加時にタイトル検索、ステータス絞り込みを追加。
- **回答URL共有の成功体験を強化**
  - コピー成功時のトースト表示 + 失敗時の代替（手動コピー用テキスト表示）。

## 3. フォーム作成・編集（管理者）

- **保存状態の可視化**
  - `保存済み` / `未保存` / `保存エラー` をヘッダーに常時表示。
- **質問テンプレートの提供**
  - 単一選択・複数選択・自由記述をワンクリック追加し、初期雛形を自動生成。
- **必須設定の一括編集**
  - 複数質問をまとめて必須化/解除できる操作を用意。
- **公開前チェック**
  - 「無題フォーム」「選択肢ゼロ」「必須なのに説明不足」などを送信前に警告。
- **プレビューの文脈一致**
  - 回答画面と同じ余白・ラベル順で表示し、作成時と実際の見え方の差異を低減。

## 4. 回答画面（回答者）

- **進捗表示（質問数/現在位置）**
  - 長いフォームで「あとどれくらいか」を表示して離脱を抑制。
- **必須項目の即時フィードバック**
  - 送信時だけでなく、入力離脱時にも「未入力」を分かりやすく表示。
- **モバイル最適化**
  - 選択肢のタップ領域を拡大し、スクロール時に送信ボタンを見失わない設計に。
- **送信前確認の簡易ステップ**
  - 送信直前に「未回答項目」「自由記述の文字数超過」を明示。
- **送信後の次アクション提示**
  - 送信完了メッセージに「閉じる」「再回答不可/可」等の案内を追加。

## 5. 集計画面（管理者）

- **一次指標の固定表示**
  - 回答件数、回答率、最新回答日時を上部に固定。
- **設問単位の見やすさ改善**
  - 選択肢比率を棒グラフ化、自由記述は最新順+キーワード抽出。
- **比較軸の追加**
  - 時系列（週次/月次）比較や、公開後経過日数に対する回答推移を表示。

## 6. 共通 UX / UI 品質

- **用語の統一**
  - `フォーム` / `アンケート` / `回答` の表記ゆれを排除。
- **アクセシビリティ改善**
  - ラベルと入力要素の関連付け、キーボード操作、コントラスト比を確認。
- **エラー表現の統一**
  - 色だけでなくアイコン・文言で意味を伝え、再操作導線を必ず併記。
- **パフォーマンス最適化**
  - 初期表示で必要な情報から描画し、集計など重い処理は段階的に表示。

## 7. 優先実装ロードマップ（短期）

1. **回答完了率の改善**
   - 必須エラーの即時表示、進捗表示、モバイルのタップ領域改善。
2. **管理者作業時間の短縮**
   - 質問テンプレート、保存状態表示、公開前チェック。
3. **運用しやすさの強化**
   - ダッシュボード検索、状態表示、集計一次指標の固定。
